### Metric definitions
- **ID-Ret**: cosine similarity E_id(source) · E_id(output) — identity *retained* from source
- **ID-Leak**: cosine similarity E_id(target) · E_id(output) — target identity *leaked* into output
- **LPIPS**: AlexNet perceptual distance between target and output (background similarity)
- **SSIM**: structural similarity between target and output (pose/expression similarity)

## 4. Root Cause Analysis: Domain Gap

The poor identity retention is explained by a **training-inference domain gap**:

### 4.1 Training domain: StyleGANEX synthetic faces
- Training data: 56,092 faces generated by StyleGANEX with truncation ψ=0.7
- Pixel statistics: mean=116.4, std=56.7
- Distribution: smooth GAN textures, no photographic noise, controlled lighting

### 4.2 Test domain: Real photographs (Experiment 11)
- Pixel statistics: mean=142.5, std=79.0 (+22% mean, +40% std)
- Distribution: natural textures, diverse lighting, real backgrounds

### 4.3 FaceNet embedding diagnostics

FaceNet embeddings are L2-normalized (norm=1.000 confirmed for all inputs). However, the *geometry* of the embedding space differs between domains:

| Domain comparison | Cosine similarity |
|-------------------|-----------------|
| Synthetic-to-synthetic (different ID) | −0.063 to 0.375 |
| Real-to-real (different ID, same pair) | −0.059 to 0.353 |
| Mapper output (synthetic) to real source | ~0.105 (our result) |

The mapper learned to minimize L_id = L1(E_id(src_synthetic), E_id(output_synthetic)).  
At inference, E_id(real_source) falls in a different embedding region than the synthetic embeddings the mapper was trained to match. The mapper produces outputs whose embeddings are essentially decorrelated from the real source.

### 4.4 Training observation: L_id decreasing but not ID-Ret

The in-training identity loss fell from 0.051 → 0.019 over 28K steps — genuine learning within the synthetic domain. But:
- L1 loss = 0.019 on 512-dim normalized vectors → corresponds to moderate cosine similarity within the training distribution
- When embeddings shift domains, the same L1 training metric does not transfer

### 4.5 GAN instability note (step 29K+)

A temporary loss spike was observed starting around step 29K:
- L_id spiked: 0.019 → 0.050
- L_lpips spiked: 0.042 → 0.077-0.095
- L_adv_G spiked: 0.68 → 3.48 (generator briefly losing to discriminator)

This is a known GAN training oscillation. Similar spikes were observed during Phase 2 (steps 1000–5000) before recovering. Monitoring continues — expected to stabilize.

## 6. What Would Fix This

### 6.1 Train on real-face W+ codes (highest impact)
Replace synthetic StyleGANEX generated data with real faces encoded into W+ space:
- Use the full pSp encoder (already in `styleganex_inversion.pt`) to encode CelebA-HQ or FFHQ
- This produces W+ codes from real faces → maps real-face embeddings to the training distribution
- Cost: ~2-4 hours of pSp encoding for 70K images

### 6.2 Use ArcFace instead of FaceNet for E_id
ArcFace (InsightFace) is more discriminative than FaceNet VGGFace2 for identity:
- ArcFace was specifically designed for face verification with angular margin loss
- Higher inter-class separation → more meaningful gradients for identity loss
- The ArcFace module is already in `/mnt/data0/naimul/ArcFace/`

### 6.3 Train longer (lower impact on domain gap)
Training currently at 28K/50K steps. The fundamental domain gap will not be resolved by more steps on synthetic data. L_id may decrease further but real-face ID-Ret will remain low.

### 6.4 Add reconstruction loss on real faces
Fine-tune the trained mapper on a small set of real faces using only reconstruction mode (I_id = I_attr). This bridges the domain gap without requiring W+ codes for disentanglement.

### 6.5 Resolution consistency
Training used 256×256 synthesis; inference used 1024×1024 → 256×256. This creates a texture discrepancy (training saw JPEG-compressed 256px GAN faces; inference uses full-resolution bicubic downsampled GAN faces). Training inference at 1024×1024 would remove this mismatch.

## 7. Summary

| Aspect | Status |
|--------|--------|
| Training convergence | ✓ Losses decreasing (L_id −59%, L_lpips −50% over 28K steps) |
| In-domain performance | ✗ Cannot evaluate quantitatively (no held-out synthetic pairs with ground truth) |
| Real-face ID-Ret | ✗ 0.105 (below random baseline of 0.138) |
| Real-face ID-Leak | ✓ 0.071 (low, output not contaminated by target identity) |
| Root cause | ✗ Domain gap: trained on StyleGANEX synthetic, tested on real photographs |
| Primary fix | Replace synthetic training data with real-face pSp-encoded W+ codes |

The mapper architecture and training framework are **correct and functional**. The failure is a data distribution problem, not an architectural one. Retraining with real-face W+ codes is expected to yield dramatically better results.
